# Spring Statemachine

Spring Statemachine (SSM) — это платформа, которая позволяет разработчикам приложений использовать 
традиционные концепции конечных автоматов с приложениями Spring.

### SSM предоставляет следующие возможности:
* Простой в использовании плоский (одноуровневый) конечный автомат для простых случаев использования.
* Иерархическая структура конечного автомата для упрощения сложной конфигурации состояний.
* Области конечного автомата для обеспечения еще более сложных конфигураций состояний.
* Использование триггеров, переходов, ограничителей и действий.
* Типобезопасный адаптер конфигурации.
* Слушатели событий конечного автомата.
* Интеграция Spring IoC для связи bean-компонентов с конечным автомато

### Проект является хорошим кандидатом на использование конечного автомата, когда:
* Вы можете представить приложение или часть его структуры в виде состояний.
* Вы хотите разделить сложную логику на более мелкие управляемые задачи.
* В приложении уже возникают проблемы с параллелизмом, например, что-то происходит асинхронно.

### Вы уже пытаетесь реализовать конечный автомат, когда:
* Используйте логические флаги или перечисления для моделирования ситуаций.
* Имейте переменные, которые имеют значение только для некоторой части жизненного цикла вашего приложения.
* Прокрутите структуру if-else (или, что еще хуже, несколько таких структур), 
проверьте, установлен ли определенный флаг или перечисление, 
а затем создайте дальнейшие исключения о том, что делать, 
когда определенные комбинации ваших флагов и перечислений существуют или не существуют.

### Три различных типа переходов: 
* external
* internal
* local
